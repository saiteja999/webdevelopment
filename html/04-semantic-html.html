FAQ Accordion with details/summary

What is semantic HTML?

Semantic HTML uses elements that describe the meaning of their content, like <article>, <nav>, and <footer>, instead of generic <div> tags.

Why does it matter?

It improves accessibility (screen readers understand the page structure), SEO (search engines parse meaning), and code readability.

Is it hard to learn?

Not at all — it's mostly about choosing the right element for the job instead of defaulting to div.
